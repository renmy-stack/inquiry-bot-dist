# 会社側で実施すること（準備チェックリスト）

inquiry-bot（問い合わせ対応アシスト）を会社で動かすために、**会社側で用意・設定が必要なもの**をまとめます。
ツール本体（exe）はPython不要でそのまま動くので、ここに挙げるのは主に **RedMine 側の設定** と **データの準備** です。

> 進め方の目安: まず **A（RedMine REST API）** と **D（Q&Aデータ）** が揃えば、基本の「チケット番号→解答案コメント」が動きます。
> **B（メール自動起票）** と **E（SVN深掘り）** は後から足せます。

---

## A. RedMine REST API を有効化（必須）

- [ ] **RedMineのバージョン**を確認（管理→情報 等）。REST API はほぼ全バージョンで利用可。
- [ ] 管理 → 設定 → **API** → 「**RESTによるWebサービスを有効にする**」にチェックして保存。
- [ ] botが使うユーザー（担当者アカウント）で **APIアクセスキー**を発行
      （右上「個人設定」→「APIアクセスキー」→【表示】）。→ GUIの「APIアクセスキー」に入れる。

## B. 対象プロジェクト（必須）

- [ ] 問い合わせを管理する**プロジェクト**を用意（既存でも可）。
- [ ] その**プロジェクト識別子**を控える（URL `.../projects/●●` の ●●）。→ GUIの「プロジェクト識別子」に入れる。
- [ ] botユーザーがそのプロジェクトの**チケット閲覧・コメント追加**権限を持つこと。

## C. 権限・アカウント

- [ ] botユーザーに、対象プロジェクトで **チケットの参照とメモ(コメント)追加** ができる権限（ロール）を付与。
- [ ] 自動投稿の主体を分かりやすくするため、**専用ユーザー**（例: `inquiry-bot`）を作るのも推奨（任意）。

## D. Q&Aデータの準備（必須）

過去の問い合わせと回答を、次のスキーマの **Excel（1行=1Q&A）** で用意します（同梱の `qa-template.xlsx` を利用）。

| 列 | 例 |
|---|---|
| ID | 1（空欄なら自動採番）|
| 受付日 | 2026-07-01 |
| 製品・システム名 | 販売管理システム |
| カテゴリ | 不具合 / 仕様確認 / 運用 など |
| 問い合わせ内容 | 月次締めでタイムアウトする |
| 回答内容 | IX_SALES_YM を追加して解消 |
| 参照設計書 | DMSF:/設計書/締め/月次締め.docx |
| 参照プログラム | svn:/trunk/sales/batch/MonthlyClose.cs |
| キーワード | 締め,タイムアウト,バッチ |
| RedMineチケット番号 | 150 |

**置き場所は2通り**（どちらか）:

- **① RedMine DMSF に置く（推奨）**
  - [ ] **DMSFプラグイン**が有効か確認（無い場合は情シスで導入検討、または②で運用）。
  - [ ] 上記ExcelをDMSFにアップロードし、その**DMSFファイルID**を控える
        （ファイルを開いたURL `.../dmsf/files/●●` の ●●）。→ GUIで「RedMine DMSF」を選び、ファイルIDを入れる。
  - [ ] botユーザーがそのDMSFファイルを**ダウンロードできる**権限を持つこと。
- **② ローカルExcelで運用（DMSFが使えない場合）**
  - [ ] Excelを `qa.xlsx` として **exeと同じフォルダ**に置く。→ GUIで「ローカルExcel」を選ぶ。

## E.（任意・Phase 2）メール→チケット自動起票

問い合わせメールを自動でチケット化したい場合。RedMine標準機能を使うので追加開発は不要です。

- [ ] 問い合わせ受付用の**メールアカウント（IMAP受信可）**を用意。
- [ ] RedMine の **IMAP取り込み**を定期実行（cron / タスクスケジューラ）。例:
      ```
      rake redmine:email:receive_imap RAILS_ENV=production \
        host=<IMAPホスト> username=<受付アカウント> password=<pw> ssl=1 \
        project=<プロジェクト識別子> tracker=<トラッカー> \
        folder=INBOX move_on_success=processed
      ```

## F.（任意・Phase 2）SVN 参照（回答の深掘り用）

- [ ] 過去回答の「参照プログラム」を実際に読んで深掘りする場合、**SVN作業コピー**を参照できる環境を用意。

## G.（任意）解答案を完全自動生成したい場合（brain=api）

- [ ] 会社で使える **LLMのAPIキー**（Anthropic もしくは OpenAI互換）を用意。→ GUIの「AI設定」に入れて解答方式を `api` に。
- [ ] 使わない場合は不要（`baseline`＝過去回答の下書き、`agent`＝Copilot等 で運用可）。

## H. 実行環境

- [ ] **Windows 10 / 11（64bit）** のPCにexe一式を配置（**Pythonのインストールは不要**）。
- [ ] 社内RedMineに**ネットワーク到達**できること（api利用時はLLMのエンドポイントにも）。

---

## 最初にやると詰まりにくい順番

1. **A**（REST API有効化＋APIキー）→ **B**（プロジェクト識別子）を用意
2. **D①/②** でQ&Aデータを配置
3. GUIの「設定」タブに入力 → **「🔌 接続テスト」** で疎通確認 → 保存
4. 「実行」タブでチケット番号を入れ **「確認のみ」** で試す → 問題なければ投稿
5. 慣れたら **E（メール自動起票）**・**G（完全自動api）** を追加

## 確認事項の最小セット（情シス／RedMine管理者へ）

- [ ] RESTによるWebサービスは有効化できるか／APIキーは発行できるか
- [ ] DMSFプラグインは使えるか（Q&Aの一元管理に利用。無ければローカルExcel運用）
- [ ] 問い合わせ用のメールアカウント（IMAP）は用意できるか（メール自動起票を使う場合）
- [ ] （完全自動を使う場合）LLMのAPIキーは用意できるか

不明点は、まず GUI の「❓ 入れ方ガイド」も参照してください（各項目の取得手順を掲載）。
