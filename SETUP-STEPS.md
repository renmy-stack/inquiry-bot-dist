# 会社側 詳細手順書（画面操作つき）

[COMPANY-SETUP.md](COMPANY-SETUP.md) の各項目を、**実際にどこを押すか**まで具体化した手順書です。
（RedMine 5 で検証。バージョンや言語設定でメニュー名が多少異なる場合があります。英語UIの表記も併記します）

- 管理者作業＝RedMineの管理者権限が必要
- 情シス作業＝サーバ側（RedMineをインストールしたマシン）での作業

---

## 手順1. REST API を有効化する（管理者作業）

1. RedMine に**管理者**でログインする。
2. 画面上部の **「管理」**（Administration）をクリック。
3. 左メニュー **「設定」**（Settings）→ 上部タブ **「API」** を開く。
4. **「RESTによるWebサービスを有効にする」**（Enable REST web service）に**チェック**。
5. 下部の **「保存」**（Save）をクリック。

> これで外部プログラム（inquiry-bot）からのAPI操作が許可されます。

---

## 手順2. APIアクセスキーを取得する（botが使うユーザーで）

1. bot が使うユーザー（担当者アカウント、または専用の `inquiry-bot` ユーザー）でログイン。
2. 画面右上の **「個人設定」**（My account）をクリック。
3. 右側の **「APIアクセスキー」**（API access key）の欄で **「表示」**（Show）をクリック。
4. 表示された英数字（例: `a1b2c3d4...`）を**コピー**しておく。→ GUIの「APIアクセスキー」に貼り付ける。

> キーが表示されない場合は、手順1が未完了です。手順1を先に実施してください。
> キーは秘密情報です。メール等で平文共有しないでください（GUIに入れれば保存時に暗号化されます）。

---

## 手順3. 対象プロジェクトと識別子を確認する（管理者作業）

**既存プロジェクトを使う場合:**
1. 対象プロジェクトを開く。
2. ブラウザのアドレスバーの URL を見る： `https://<RedMine>/projects/support` の **`support`** の部分が**識別子**。
   （プロジェクトの「設定」→「情報」の「識別子」欄でも確認可）→ GUIの「プロジェクト識別子」に入れる。

**新規に作る場合:**
1. **「管理」→「プロジェクト」→「新しいプロジェクト」**（New project）。
2. 名称（例: 問い合わせ対応）と**識別子**（半角英数、例: `support`）を入力。
3. 下部の **「モジュール」** で **「チケットトラッキング」**（Issue tracking）にチェック。
4. **「作成」**（Create）。

---

## 手順4. bot ユーザーの権限を設定する（管理者作業）

bot ユーザーが対象プロジェクトで**チケットを見てコメントを付けられる**ようにします。

1. **「管理」→「ロールと権限」**（Roles and permissions）。
2. bot ユーザーに割り当てるロール（例: 「開発者」）を開く。
3. **「チケットトラッキング」** の項目で、少なくとも次にチェックが入っていること：
   - **チケットの閲覧**（View issues）
   - **メモの追加**（Add notes）
4. 保存。
5. **「プロジェクト」→ 対象プロジェクト →「設定」→「メンバー」→「新しいメンバー」** で、
   bot ユーザーを上記ロールで**メンバー追加**。

---

## 手順5. Q&A Excel を用意する

同梱の **`qa-template.xlsx`** を開き、1行に1件で過去の問い合わせと回答を入力します。

| 列 | 入れるもの | 例 |
|---|---|---|
| ID | 空欄でOK（自動採番）| |
| 受付日 | 日付 | 2026-07-01 |
| 製品・システム名 | 対象システム | 販売管理システム |
| カテゴリ | 種別 | 不具合 / 仕様確認 / 運用 |
| 問い合わせ内容 | 質問の要約 | 月次締めでタイムアウトする |
| 回答内容 | 回答の要約 | IX_SALES_YM を追加して解消 |
| 参照設計書 | 設計書の場所 | DMSF:/設計書/締め/月次締め.docx |
| 参照プログラム | プログラムの場所 | svn:/trunk/sales/batch/MonthlyClose.cs |
| キーワード | 検索に効く語 | 締め,タイムアウト,バッチ |
| RedMineチケット番号 | 元チケット | 150 |

> 「問い合わせ内容」「キーワード」「製品・システム名」が、類似検索の精度に効きます。
> ヘッダ行（1行目）は変更しないでください。数十〜数百件で十分に機能します。

## 手順6-A. Q&A を RedMine DMSF に置く（推奨・DMSFが使える場合）

### 6-A-1. DMSF が使えるか確認（管理者作業）
1. **「管理」→「プラグイン」**（Plugins）を開く。
2. 一覧に **「DMSF」**（Redmine DMSF）があればインストール済み。無ければ 6-A-0 を実施。
3. 対象プロジェクトの **「設定」→「モジュール」** で **「DMSF」** にチェックして保存。

### 6-A-0.（DMSFが無い場合）DMSFプラグイン導入（情シス作業・任意）
DMSFはRedMine標準ではなく**追加プラグイン**です。サーバ側で導入します（一般的な手順）：
```
# RedMineのプラグインフォルダへ
cd <REDMINE>/plugins
git clone https://github.com/danmunn/redmine_dmsf.git redmine_dmsf
cd <REDMINE>
bundle install
bundle exec rake redmine:plugins:migrate RAILS_ENV=production
# RedMine(またはWebサーバ)を再起動
```
> 導入可否・バージョン整合は情シス判断で。難しければ **手順6-B（ローカルExcel）** で運用できます。

### 6-A-2. Q&A Excel をアップロードして「ファイルID」を確認
1. 対象プロジェクトの **「DMSF」** タブを開く。
2. **「ファイルを追加」/アップロード** で `qa.xlsx` をアップロード。
3. アップロードしたファイル名をクリックして開く。
4. アドレスバーの URL `https://<RedMine>/dmsf/files/12/...` の **`12`** が **DMSFファイルID**。
   → GUIで「RedMine DMSF」を選び、この数字を入れる。

---

## 手順6-B. Q&A をローカルExcelで置く（DMSFを使わない場合）

1. 記入済みの Excel を **`qa.xlsx`** という名前にする。
2. `inquiry-bot-gui.exe` と**同じフォルダ**に置く。
3. GUIで「ローカルExcel」を選び、ファイル名 `qa.xlsx` を入れる。

---

## 手順7. GUI で設定して動作確認

1. `inquiry-bot-gui.exe` をダブルクリックで起動。
2. **「設定」タブ**に入力：
   - RedMine URL（例 `https://redmine.example.local`）
   - APIアクセスキー（手順2でコピーしたもの）
   - プロジェクト識別子（手順3）
   - Q&Aデータの場所（手順6-A または 6-B）
3. **「🔌 接続テスト」** を押す →「接続OK: プロジェクト『…』」が出れば成功。
   - 失敗時は末尾の「困ったときは」を参照。
4. **「💾 設定を保存」**。
5. **「実行」タブ**でチケット番号を入れ、**「確認のみ」にチェックしたまま「▶ 実行」**。
   - 表示された解答案の内容が妥当そうなら、「確認のみ」を**外して**再度「▶ 実行」＝実投稿。
6. RedMineの該当チケットを開き、🤖 のコメントが付いていれば完了。

## 手順8.（任意）メール→チケット自動起票（Phase 2）

問い合わせメールを自動でチケット化します（RedMine標準機能・追加開発不要）。

### 8-1. 受付メールアカウント
- 問い合わせ受付用の**メールアドレス（IMAP受信可）**を用意（情シス）。

### 8-2. 取り込みコマンドの確認（情シス・サーバ側）
```
cd <REDMINE>
bundle exec rake redmine:email:receive_imap RAILS_ENV=production \
  host=<IMAPホスト> port=993 ssl=1 \
  username=<受付アカウント> password=<パスワード> \
  project=<プロジェクト識別子> tracker=<トラッカー名> \
  folder=INBOX move_on_success=processed move_on_failure=failed
```
一度手動で実行し、受信箱のメールがチケット化されることを確認。

### 8-3. 定期実行（Windowsサーバの例：タスクスケジューラ）
1. **タスクスケジューラ** を開く →「基本タスクの作成」。
2. トリガー：例「毎日」→「1回」→「繰り返し間隔 5分」等。
3. 操作：上記コマンドを実行するバッチ（.bat）を指定。
4. 保存。以後、新着メールが自動でチケット化されます。

> Linuxサーバなら cron に同コマンドを登録（例: `*/5 * * * *`）。

## 手順9.（任意）解答案を完全自動生成する（brain=api）

1. 会社で使える **LLMのAPIキー**を用意（Anthropic もしくは OpenAI互換）。
2. GUIの「設定」タブ →「解答方式(brain)」を **`api`** に。
3. 「AI設定」で provider（anthropic/openai）・model（例 `claude-sonnet-5`）・AI APIキーを入力。
4. 保存 →「実行」で試す（まず「確認のみ」で生成内容を確認）。

> 使わない場合は不要。`baseline`（過去回答の下書き）や `agent`（Copilot等）で運用できます。

## 困ったときは（接続テストが失敗する）

| 表示・症状 | 主な原因と対処 |
|---|---|
| 認証エラー | APIキーが違う／手順1のREST API有効化が未実施。手順1・2をやり直す。|
| プロジェクトが見つからない | プロジェクト識別子の誤り（URLの `.../projects/●●`）。手順3で再確認。|
| 接続失敗（つながらない）| URLの誤り、社内ネットワーク未到達、RedMine停止。URLとネットワークを確認。|
| Q&Aが0件 | DMSFファイルIDの誤り／botユーザーにDMSFダウンロード権限が無い／ローカルExcelの置き場所・ファイル名違い。|
| 投稿されない（確認のみのまま）| 「確認のみ」のチェックを外して実行。|

各項目の入れ方は、GUI設定タブ右上の **「❓ 入れ方ガイド」** でも確認できます。
